问题定义

有n个人，第0个人从1开始报数，报到m的人出列，然后从下一个人开始报数。求最终剩下人的序号

各种解法

直白的暴力求解

思路

  • 定义一个n长的数组array，初值都为1，表示都还活着
  • 定义一个索引index
    1. 它从0到n-1自增，当变为n时，回到0，也就是(++index) %n
    2. 它跳过array[index]为0的那些项
  • 定义一个计数器sum，从0开始计数，到n时推出循环，表示当前有多少人出列了。
  • 定义一个计数器count
    1. 它从0计数到m
    2. 当变为m时，将array[index]置为0，表示被移除。将sum自增

分析

这种方式，逻辑上还是比较清晰的，题目怎么来我就怎么算。缺点也很明显

  1. %n不高效
  2. 当n很大时，数组持续占据很大内存，而且计算最后一个人时，即使其他人都”死了”，却还要再重新遍历一遍

优雅地利用数据结构

某人报到号出队时，需要将它移出原数据。这时数组显然不合适，自然想到了链表。

既然所有的人是一个圈，即最后一个人的下一个人是第一个人，那自然使用循环链表

使用单向循环链表还是双向循环链表呢，报号是从前往后的，理论上不需要back。因此使用单项循环链表实现。

ps. 今年南大AI初试里，有道填空题就考到了循环链表解决约瑟夫问题

思路

  • 建立一个单向循环链表，每个节点存有的数据即为索引
  • 报数即为指向下一节点
  • 链表里只剩一项时即为结果

需要注意的点是，由于是单链表，至少需要前后两个指针才能做到删除节点

巧妙的另辟蹊径

由于最终结果只是要求出，最终剩下人的编号，是不是可以跳过移除的逻辑，直接求得呢？

参考约瑟夫环之二(用递归的思想解决Josephus问题)

思路

  1. 假设我们有一个神奇的函数f(n)，它能最终返回剩下的人的编号

  2. 首先我们知道，f(1)为0。即只有1个人的时候，剩下的即是第0号

  3. 怎么得到f(n)和f(n-1)的关系呢？啊哈，在有n个人的时候，把第一个人先T掉，不就只剩下n-1个人了嘛，这样我们就朝目标迈进了一步啦。

    • 我们要知道在有n个人的时候，最先被T的人的编号

      • 当n比较大m比较小的时候，很容易知道这个编号是m-1。比如10个人报3号的时候，最先走的就是3-1=2号
      • 那当n<m的时候呢，这时候可以设m=k*n+i (0<i<=n)，则最先走的是i-1号，而(i-1)%n == (m-1)%n
      • 综合两种情况，最先被T的人的编号就是(m-1)%n

    • 然后我们要知道被T的人的下一个人的编号。它显然是((m-1)%n+1)%n = m%n，设为k

    • 然后知道，n-1个人的时候

      • 如果从第0个人开始报数，最终剩下的人编号即为f(n-1)，这是我们最开始的假设
      • 如果从第k个人开始报数，最终剩下的人编号即为f(n)。因为这只是我们在求f(n)时，把第一个人T掉的余下的过程。

      • 而从第k个人开始报数的结果，它等于从第0个人开始的结果加上k(要模n防止溢出)。因为相当于每个报数的人”基数”增加了k。第0个相当于第k个，第一个相当于第k+1个…

        

- 因此`f(n) = (f(n-1) + k) % n = (f(n-1)+m%n)%n = (f(n-1)+m)%n`
