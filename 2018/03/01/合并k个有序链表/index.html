准备工作

  • 单链表的定义、创建、输出等，老生常谈了

单链表的创建用了vector，也可以不用的，这里为了方便，不过这不是重点;
链表的头结点也不作为特殊节点了，直接作为数据节点。

  • 合并两个有序单链表

也好理解，和归并排序有点像。

下面两个合并k个链表的算法都需要用到该方法。

算法一：两两合并

这个好理解的，当链表不为空时：L[0]和L[1]合并->再和L[2]合并->…

算法二：各种巧妙合并

以合并7个链表为例

  • 第一种方法

第一轮，size为7

  1. L[0]和L[1]合并到L[0]
  2. L[2]和L[3]合并到L[1]
  3. L[4]和L[5]合并到L[2]
  4. L[6]不合并，直接赋值到L[3]
    注意到经过上面的过程之后，需要再合并的链表的长度减半了，即变为了(7+1)/2=4

第二轮，size为4

  1. L[0]和L[1]合并到L[0]
  2. L[2]和L[3]合并到L[1]

第三轮，size为2

  1. L[0]和L[1]合并到L[0]
    结束退出

  • 第二种方法
    以递增的间隔合并

第一轮，间隔为1

  1. L[0]和L[1]合并到L[0]
  2. L[2]和L[3]合并到L[2]
  3. L[4]和L[5]合并到L[4]
  4. L[6]不动

第二轮，间隔加倍为2

  1. L[0]和L[2]合并到L[0]
  2. L[4]和L[6]合并到L[4]

第三轮，间隔加倍为4

  1. L[0]和L[4]合并到L[0]

间隔再加倍超过7时退出

  • 第三种方法
    以递减的间隔合并（这个有点像希尔排序的思想了）

第一轮，间隔为(7+1)/2=4

  1. L[0]和L[4]合并到L[0]
  2. L[1]和L[5]合并到L[1]
  3. L[2]和L[6]合并到L[2]
  4. L[3]不做修改
    如此之后，size减半为4，只需要管L[0..3]了

第二轮，间隔为(4+1)/2=2

  1. L[0]和L[2]合并到L[0]
  2. L[1]和L[3]合并到L[1]
    size减半为2，只需要管L[0..1]了

第三轮，间隔为(2+1)/2=1

  1. L[0]和L[1]合并到L[0]
    结束，返回L[0]

算法三：利用优先队列

C++ priority_queue优先队列的用法参考
大致的思路如下：

  1. 每次都以所有链表的head元素作为比较元素，建立最小堆。即队列的top的head元素是最小的
  2. 将top元素初队列，取出head元素（此时为所有链表最小的元素），head指向head->next
  3. 此时如果head为空了，就舍弃；否则再将head入队列
  4. 重复上面的步骤，直到队列为size为1。则直接将剩下的元素添加到结果链表尾部即可
